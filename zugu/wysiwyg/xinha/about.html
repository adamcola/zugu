[Security] Systems have been protected by a securely encrypted code, meaning that even with the correct hacking skills the site stays secure. The second wall of protection will stop injected scripts from working as the only code that is recognized has been securely protected.

[Defense] To further strengthen defenses against attack the site is regularly backed up. All files are mirrored in the event that the server fails.

[Attack] Although we are keen to strengthen our methods of security if there are any weaknesses, any attempts to breach our systems will be treated very seriously. We would prefer a simple email warning relating to any security issues rather than a "friendly hack", as any i.p address not from within the university will be treated as a breach in the data protection act and handed over to the correct authorities.
